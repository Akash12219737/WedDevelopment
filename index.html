Internet Basics

  1. Principles of the Internet

    • Need to Share Information:- Computers exchange data through direct links (cables, signals of 1s & 0s).
    • Problem of Scale:- Many-to-many links become messy → solved by Switches (Local Networks).
    • Connecting Networks:- Multiple local networks connect using Routers → forms a network of networks (Internet).
    • Finding Devices (Addressing):- Each device needs a unique IP Address to communicate.
    • Reliability & Fairness: Data broken into packets with sender/receiver info, reassembled via TCP/IP protocol.

  2. Ip Address

    • IPv4: 32-bit (e.g., 172.217.16.142) → ~4.3 billion addresses → shortage problem.
    • Public IP: Globally unique, assigned by ISP, used to access internet.
    • Private IP: Used inside local networks (192.168.x.x, 10.x.x.x, 172.16-31.x.x).

  3. MAC Address

    • Unique 48-bit hardware ID of a device's network card.
    • Written in hex (e.g., 3C:22:FB:A3:B4:C5).
    • Permanent, unlike IP which can change.
    • Found via commands (ipconfig /all, getmac /v, ifconfig).

  4. Port Numbers

    • 16-bit identifier (0–65535) to specify services.
    • Well-Known Ports (0–1023): Standard services (HTTP 80, HTTPS 443, DNS 53, SSH 22).
    • Registered Ports (1024–49151): Used by apps/databases (Node.js 3000, MySQL 3306, MongoDB 27017).
    • Dynamic Ports (49152–65535): Temporary client-side connections.

  5. DNS (Domain Name System)

    • Converts domain names → IP addresses.
    • Process:
      1. Check local cache (browser, OS, router).
      2. Ask Recursive Resolver (usually ISP or public like Google 8.8.8.8).
      3. Resolver queries DNS hierarchy:
        • Root Servers → point to TLD servers (e.g., .com)u
        • TLD Servers → point to Authoritative servers.
        • Authoritative Server → gives final IP.
      4. Resolver caches response & sends IP to browser.
      5. Browser connects to IP (usually via port 80/443) and loads the website.
