Boilerplate
  • In web development, boilerplate means the basic, standard code template that every HTML page usually starts with.
  • It gives your web page a proper structure and ensures browsers understand it correctly.

Think of it like a foundation of a house - before you decorate or add things, you need the basic frame.

Explanation of Each Part

Part Code Element Purpose & Details
1 <!DOCTYPE html>
  • Tells the browser that this is an HTML5 document.
  • Ensures the page is rendered in standard mode (not old compatibility mode).
2 <html lang="en">
  • Root element of the page.
  • lang="en" tells the browser (and search engines, screen readers) the page is in English.
3 <head> Section
  • <meta charset="UTF-8"> Ensures the page supports all characters (like English, Hindi, emojis ).
  • <meta name="viewport" content="width=device-width, initial-scale=1.0">
4 <body> Section
  • Contains the content visible to users (text, images, videos, links, etc.).
  • Example: headings, paragraphs, forms, buttons, etc.

Why is Boilerplate Important?

  • Saves time (you don't have to write the basic structure every time).
  • Makes your page work correctly in all browsers.
  • Ensures SEO-friendly and responsive pages.

ID in HTML
  • ID is also an attribute, but it should be unique (used only once per page).
  • It is used when you want to target one specific element for styling or JavaScript.

Quick Summary

Attribute When to Use Examples
Class You want to apply the same style to multiple elements. All buttons, all paragraphs, all sections.
ID You want to style or target one unique element. The navbar, footer, or one special heading.
