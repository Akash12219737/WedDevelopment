Internet Basics - Complete Guide

1. Principles of the Internet

Concept Explanation
Need to Share Information Computers exchange data through direct links (cables, signals of 1s & 0s)
Problem of Scale Many-to-many links become messy → solved by Switches (Local Networks)
Connecting Networks Multiple local networks connect using Routers → forms a network of networks (Internet)
Finding Devices (Addressing) Each device needs a unique IP Address to communicate
Reliability & Fairness Data broken into packets with sender/receiver info, reassembled via TCP/IP protocol

3. MAC Address (Hardware Identifier)

Key Features:

  • Unique 48-bit hardware ID
  • Written in hexadecimal format
  • Permanent (unlike IP which can change)
  • Identifies network card physically

Example & Commands:

Format: 3C:22:FB:A3:B4:C5

Find MAC Address:

  • ipconfig /all (Windows)
  • getmac /v (Windows)
  • ifconfig (Linux/Mac)

4. Port Numbers (Service Identifiers)

Definition: 16-bit identifier (0–65535) to specify services running on a device

Port Range Purpose Common Examples
Well-Known Ports
(0–1023)		 Standard system services
  • HTTP: 80
  • HTTPS: 443
  • DNS: 53
  • SSH: 22
Registered Ports
(1024–49151)		 Applications & databases
  • Node.js: 3000
  • MySQL: 3306
  • MongoDB: 27017
Dynamic Ports
(49152–65535)		 Temporary client connections Auto-assigned by system

5. DNS (Domain Name System)

Primary Function: Converts domain names → IP addresses

Complete DNS Resolution Process:

Step Action Detailed Process
1 Check Local Cache Browser → OS → Router cache lookup
2 Ask Recursive Resolver Usually ISP or public DNS (Google 8.8.8.8)
3 Query DNS Hierarchy
  1. Root Servers → point to TLD servers (e.g., .com)
  2. TLD Servers → point to Authoritative servers
  3. Authoritative Server → gives final IP address
4 Cache & Return Resolver caches response & sends IP to browser
5 Connect & Load Browser connects to IP (via port 80/443) and loads website

DNS in Action Example:

You Type:
www.google.com
DNS Magic		 Browser Gets:
142.250.191.14
